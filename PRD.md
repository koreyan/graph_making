# Product Requirements Document (PRD): 리플로우 프로파일 데이터 분석 및 어노테이션 도구 (v2.0)

## 1. 프로젝트 개요 (Project Overview)
본 프로젝트는 반도체 및 SMT(Surface Mount Technology) 공정 엔지니어가 온도 프로파일 데이터를 정밀하게 분석하고 보고서용 주석을 관리할 수 있는 고성능 데스크톱/웹 애플리케이션을 구축하는 데 목적이 있습니다.

기존 v1.0의 단순한 수동 그래프 렌더링 및 드래그 계산 명세에서 벗어나, 반도체 리플로우 도메인 전문 가이드(`REFLOW_PROFILE_GUIDE.md`)를 완벽히 반영하여 **'5대 핵심 구간의 자동 물리 수치 계산 엔진'**과 **'솔더별 실시간 안전 규격 검증 및 경고(Safety Margin Check) 시스템'**을 내장한 프리미엄 '엔지니어링 인사이트 도구'로 거듭납니다.

---

## 2. 기술 스택 (Tech Stack)
현대적인 UI와 정밀한 엔지니어링 차트 인터랙션을 구현하기 위해 검증된 고성능 라이브러리를 채택합니다.

| 분류 | 기술 | 선정 이유 |
| :--- | :--- | :--- |
| **Framework** | **Electron** (데스크톱) / **Vite + React** (웹) | 크로스 플랫폼 호환성 확보 및 풍부한 웹 에코시스템 활용 |
| **Frontend** | **React + TypeScript** | 점진적 상태 관리 및 강력한 정적 타입을 통한 데이터 모델 안정성 확보 |
| **Visual** | **Highcharts.js** (+ Annotations, Exporting 모듈) | 엔지니어링 수준의 줌(Zoom), 마우스 인터랙션, 드래그형 주석 모듈 및 가이드라인 내장 |
| **Data** | **PapaParse** | 대용량 멀티채널 CSV 시계열 온도 데이터의 지연 없는 파싱 지원 |
| **Styling** | **Vanilla CSS (CSS Module)** | 프레임워크 제약이 없는 프리미엄 다크 모드 및 미세 애니메이션 최적화 구현 |

## 3. 핵심 기능 요구사항 (Functional Requirements)

### 3.1 스마트 단일 채널 데이터 파싱 (Priority: High)
*   **비정형 헤더 스마트 스킵**: CSV 파일 상단부의 기기/공정 메타데이터 영역(보통 Row 1~7)을 동적으로 감지하여 스킵하고 실제 데이터 필드가 시작되는 지점부터 견고하게 파싱합니다.
*   **단일 핵심 채널 추출 및 분석 (Single-Channel Focus)**:
    *   CSV 파일에 다채널 온도 데이터가 포함되어 있더라도, 엔지니어가 분석하고자 하는 단 하나의 주 열전대 채널(기본값: **'열전지 2 / TC2'**)만을 추출하여 차트에 시각화하고 분석합니다.
    *   화면에 표시되는 데이터를 단일 선으로 정제하여 멀티채널에 따른 눈의 피로도와 혼선을 방지하고, 구간 분석과 마진 체크의 정밀도에 온전히 집중하게 만듭니다.
*   **센서 이상치 및 단선 처리**: 온도 값이 `-270°C` 이하인 경우 센서 미연결 또는 단선 상태로 판정하여 그래프 렌더링에서 예외 처리(NaN 또는 null 처리)함으로써 스케일 왜곡을 방지합니다.
*   **시간 축 경과 시간(Normalized Time) 자동 변환**: 날짜/시각 포맷의 원본 데이터를 분석 개시 지점인 `0초`부터 누적 경과된 초(sec) 데이터로 정규화하여 X축에 표시합니다.

### 3.2 고성능 인터랙티브 차트 및 자동 공정 분석 (Priority: High)

#### ① 솔더 타입 스펙 프리셋 및 동적 공정 조건 입력 (Solder Profile & Custom Inputs)
다양한 작업 환경과 조건을 손쉽게 기입하고 실시간 차트 분석에 반영할 수 있도록 **'사이드바 입력 폼(Interactive Form) + 원클릭 프리셋'** 결합 방식을 제공합니다. 사용자가 값을 직접 타이핑하거나 조절하면 차트의 가이드라인 및 안전 마진 검증 알고리즘이 실시간으로 재바인딩됩니다.

*   **원클릭 표준 프리셋 (Solder Preset Selector)**:
    *   **Pb-Free 무연 솔더 (SAC305) 프리셋 (기본값)**: 액상선(217°C), 예열소크구간(150°C~200°C), TAL 적정 범위(45~90초), 최고 온도(235°C~250°C), 냉각 속도(-1.0°C/s~-4.0°C/s) 설정값 즉시 바인딩.
    *   **Leaded 유연 솔더 (Sn63/Pb37) 프리셋**: 액상선(183°C), 예열소크구간(100°C~150°C), TAL 적정 범위(60~90초), 최고 온도(210°C~230°C), 냉각 속도(-1.0°C/s~-4.0°C/s) 설정값 즉시 바인딩.
*   **실시간 공정 조건 커스텀 입력 패널 (Custom Process Inputs Panel)**:
    *   **솔더 성분/화학 조성비 직접 기입**: 예: `Sn 96 / Ag 3.5 / Cu 0.5` 또는 `SAC305` 등 텍스트 박스를 활용한 커스텀 솔더 명칭 기입 (보고서 헤더 및 PDF 리포트 인쇄 시 자동 표기).
    *   **컨베이어 이동 속도 (Conveyor Speed) 입력**: 리플로우 기기를 통과하는 벨트 속도(예: `mm/s` 또는 `cm/min` 단위) 기입란 제공 (엔지니어가 시간-챔버 물리적 거리 분석 및 보고용으로 기입).
    *   **임계 온도/시간 동적 오버라이드 (Dynamic Parameter Override)**: 프리셋 값 외에도 엔지니어가 액상선 온도(예: `221°C`), 소크 시작/끝 온도, 피크 온도 임계 규격 등을 화면에서 직접 수정할 수 있으며, 수정 시 **차트의 가이드라인(plotLines/plotBands)과 안전 마진 검증(Margin Check) 연산 알고리즘이 즉각 동적으로 반영**됩니다.
    *   **결과 보존과의 연계**: 수동 기입된 화학 조성비와 이동 속도는 최종 '그래프 이미지 캡처' 및 '종합 PDF 보고서' 영역에 레이블 형태로 임베딩되어 최종 문서 보존 시 활용됩니다. (별도의 JSON 설정 파일 로드는 제외하여 관리 요소를 시각 이미지로 일원화)

#### ② 5대 핵심 공정 구간 자동 계산 엔진 (Auto-Analysis Engine)
데이터 로딩 즉시 메인 채널 시계열 데이터를 역추적하여 다음 물리량들을 실시간 자동 산출합니다.

1.  **초기 승온 기울기 (Ramp-up Rate, °C/s)**:
    *   공식: $\text{Slope}_{\text{ramp-up}} = (T_{\text{SoakStart}} - T_{\text{start}}) / (t_{\text{SoakStart}} - t_{\text{start}})$
    *   *적정 기준: 1.0 ~ 3.0 °C/sec (권장 2.0 이하)*
2.  **예열 소크 유지 시간 (Soak Duration, sec)**:
    *   공식: $\text{Duration}_{\text{soak}} = t_{\text{SoakEnd}} - t_{\text{SoakStart}}$
    *   *적정 기준: 60 ~ 120 sec*
3.  **리플로우 승온 기울기 (Reflow Ramp-up Rate, °C/s)**:
    *   공식: 소크 구간이 끝나는 시점부터 최고 피크 온도 도달점까지의 평균 온도 상승률 계산.
    *   *적정 기준: 1.0 ~ 2.0 °C/sec*
4.  **TAL (Time Above Liquidus, sec)**:
    *   공식: 온도가 액상선 온도($217^\circ\text{C}$ 또는 $183^\circ\text{C}$) 이상으로 유지되는 시간 ($t_{\text{out}} - t_{\text{in}}$).
    *   *적정 기준: 무연 45 ~ 90s / 유연 60 ~ 90s*
5.  **최고 온도 (Peak Temperature, °C)**:
    *   공식: 전체 시계열 중 최고 온도 점 $\max(T_i)$ 및 해당 시간($t_{\text{peak}}$).
    *   *적정 기준: 무연 235 ~ 250°C / 유연 210 ~ 230°C*
6.  **하강 냉각 속도 (Cooling Rate, °C/s)**:
    *   공식: 피크 온도 도달 시간($t_{\text{peak}}$)부터 안전 하강 온도($100^\circ\text{C}$ 이하)까지의 평균 하강 기울기.
    *   *적정 기준: -1.0 ~ -4.0 °C/sec (권장 -3.0)*

#### ③ 실시간 규격/마진 체크 & 경고 시스템 (Safety Margin Check)
*   위 5대 구간의 계산 결과가 선택된 솔더 타입별 프리셋 기준치에 부합하는지 실시간 검증합니다.
*   **불량 마진 이탈 시(Warning)**: 대시보드 스코어카드에 **경고 기호(⚠️)**와 붉은색 알림을 즉시 점등하며, 가이드에 의거한 **불량 요인 및 현상**(예: *툼스톤 현상 유발*, *열 충격 크랙 위험*, *냉납 우려*, *부품 탄화 위험* 등)을 동적으로 화면에 안내합니다.

#### ④ 동적 가이드라인 및 배경 영역화 (Dynamic plotLines & plotBands)
*   **배경 영역화 (`plotBands`)**: X축 기준으로 Ramp-Up 영역, Preheat/Soak 영역, Reflow(TAL) 영역을 연하고 조화로운 파스텔 톤 파란/주황/빨간색 배경 밴드로 하이라이트 처리하여 한눈에 식별 가능하게 합니다.
*   **임계 가이드라인 (`plotLines`)**: Y축 기준으로 액상선 온도(217°C/183°C) 및 소크 경계 온도(150°C/200°C 등)에 점선 가이드라인과 텍스트 태그를 표시합니다.

#### ⑤ 유연한 축 설정 및 그리드 제어 (Flexible Scale Mode)
*   **축 및 기본 그리드**: 고대비 가시성을 위해 X축 30초 단위 간격, Y축 50도 단위 간격의 세밀한 보조 격자(Grid lines)를 렌더링합니다.
*   **스케일 모드 토글 (조정 사항)**:
    *   *Standard Mode (기본)*: X축 0~480초, Y축 0~300도로 고정하여 표준적 프로파일 형상을 한눈에 조망.
    *   *Auto-Fit Mode (동적)*: 파일 로딩 시 데이터의 최대 시간과 온도를 자동 분석하여 스케일 한계를 자동으로 리사이징해 잘림 현상을 완전 방지합니다.

#### ⑥ 드래그&드롭 자유 주석(Annotations) 시스템
*   기존 v1.0의 제한적인 수동 드래그 박스 방식에서 탈피하여, `Highcharts-Annotations` 모듈을 도입합니다.
*   엔지니어가 마우스로 그래프의 임의의 점을 클릭해 **'설명 풍선(Annotation Box)'**을 생성하고 텍스트 메모를 직접 기입할 수 있습니다.
*   생성된 주석은 마우스 드래그를 통해 차트 내에서 자유롭게 이동(xy-draggable)할 수 있으며, 주석 박스 삭제 및 수정 기능도 함께 지원합니다.

### 3.3 분석 결과 시각적 보존 및 리포트 내보내기 (Priority: High)
*   **보고서용 고화질 그래프 이미지 저장**: 커스텀 주석(Annotations), 온도 및 시간 가이드라인(plotLines), 공정 구간 하이라이트(plotBands)가 모두 온전히 포함된 차트 영역을 사용자의 로컬 디스크에 즉시 고화질 이미지(PNG 또는 JPEG)로 캡처하여 내려받습니다. (최종 보존 시 오직 그래프 이미지만 독립적으로 활용)
*   **종합 PDF/HTML 분석 보고서 출력 (Medium)**: 캡처된 그래프 이미지 + 5대 핵심 물리 수치 분석 테이블 + 사이드바에서 기입한 커스텀 공정 조건(화학 조성 명칭, 이동 속도) + 마진 검증 결과 경고가 정돈된 1페이지 분량의 통합 엔지니어링 리포트를 PDF 문서로 즉시 다운로드하거나 인쇄할 수 있도록 합니다.

## 4. 실제 CSV 데이터 명세 및 파싱 규칙 (Data Specification)
디렉토리에 위치한 실제 리플로우 측정 파일(`T15168 MultiChannel.csv` 및 `TEST_REFLOW_DATA.csv`)의 스펙을 완벽하게 지원합니다.

*   **원본 파일 포맷**: 표준 Comma-Separated Values (CSV) 구조
*   **헤더 및 메타데이터 자동 추출**:
    *   **Row 1~4 (메타 영역)**: `기기 이름:,,TCTempX4`, `기기 설명,,...`, `일련 번호:,,T15168`, `기기 ID:,,MultiChannel` 등의 기기 식별 정보를 정확히 건너뛰어 파싱하되, 해당 값들을 추출하여 애플리케이션 상단이나 보고서 헤더 메타 영역에 자동으로 뿌려줍니다.
    *   **Row 5**: 빈 행 (스킵)
    *   **Row 6**: 채널 이름 보조 정보 (`,,채널 2,채널 4,채널 6,채널 8`)
    *   **Row 7 (한글 데이터 헤더)**: `날짜,시간,열전지 1 (°C),열전지 2 (°C),열전지 3 (°C),열전지 4 (°C),,,,` 구조를 한글 헤더 기준으로 정밀 인식합니다.
*   **시간 포맷 정규화 및 누적 계산**:
    *   **시간 형태**: `PM 8:13:19` 또는 `AM 8:13:19` 형식으로 앞에 AM/PM 기호가 한 칸 공백으로 띄워져서 표기됩니다.
    *   **누적 시간 정규화(Normalized Cumulative Time)**: 날짜와 AM/PM 시각 문자열을 초 단위 타임스탬프로 환산하여 첫 데이터 행을 `0초`로 기준으로 잡은 뒤, 데이터 포인트마다 `1초` 단위의 경과 초(Normalized Seconds, e.g., 0, 1, 2, 3...) 데이터로 정규화하여 차트의 X축에 적층 매핑합니다.
*   **단일 채널 필터링 및 단선 감지**:
    *   **주 분석 대상**: CSV 내 여러 열 중, 실제 유효한 리플로우 온도 곡선이 온전히 시계열로 변동하는 **'열전지 2 (°C)' (TC2)** 채널만을 추출하여 분석용 주 타겟 채널로 적용합니다.
    *   **단선 및 노이즈 예외 처리**: '열전지 1', '열전지 3', '열전지 4' 등 센서가 단선되어 온도 값이 `-270`으로 채워진 채널 및 노이즈 값들은 필터링 로직을 통해 분석 및 그래프 렌더링 대상에서 완벽히 스킵 및 배제하여 불필요한 차트 스케일 오류를 사전 차단합니다.

## 5. UI/UX 디자인 전략 (Premium Dark Theme)
*   **Dark & Professional Theme**:
    *   배경색 `#121214` 및 `#1e1e24` 카드 레이아웃 적용. 
    *   온도 차트는 딥 블랙/다크 그레이 배경을 활용하여 오랜 모니터링 시에도 눈의 피로감을 차단하고 신호선의 가독성을 극대화합니다.
*   **3단 레이아웃 (High-Efficiency Console)**:
